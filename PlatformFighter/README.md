# PlatformFighter — Phase 1-4 Foundation

This project was generated by `build_project.py`. It's a buildable Godot 4.5
(.NET) foundation for a competitive platform fighter: Git hygiene, a clean
folder layout, deterministic Q32.32 fixed-point math, a 60 Hz simulation
loop fully decoupled from rendering, a deterministic multi-device input
pipeline, and a fixed-point AABB collision engine — all ready for replays
and rollback netcode down the line. See `Docs/ROADMAP.md` for phase status.

## First-time setup

1. Install **Godot 4.5+ (.NET / C# edition)** and the **.NET 8 SDK**.
2. Open this folder in Godot via *Import*. On first open with a C# project,
   Godot generates `PlatformFighter.sln` and the `.godot/` cache
   automatically — that's expected and is not tracked in Git.
3. Press **Run** (F5). `Main.tscn` is already set as the main scene.

## What you should see

A sprite bouncing horizontally between two bounds, plus a debug label
showing the sim frame counter, render FPS, ticks-per-render-frame, the
render interpolation alpha, and the local player's currently-held input.
Press **F9** to run the determinism/collision acceptance test suite —
its output prints into that same debug label and to the console.
See `Docs/DETERMINISM.md` for how to use this as a regression check and
for the ground rules every later phase must obey, `Docs/INPUT.md` for
device/key bindings and the input architecture, and `Docs/COLLISION.md`
for the AABB collision engine (not yet wired into the visible scene —
today it's exercised only through the F9 test rig; a real player body
replaces `TestMover` in Phase 5).

## Controls

WASD/Arrows to move, J to attack, K for special, W/Space to jump, L to
shield. Full details in `Docs/INPUT.md`.

## Layout

```
res://
├── Core/          # Engine-level, game-agnostic systems (Sim, Math, Input)
├── Gameplay/       # Fighter-generic logic: states, hitboxes, damage
├── Characters/      # Per-character data & scripts
├── Stages/           # Stage geometry, blast zones, spawn data
├── UI/                # Menus, HUD
├── Networking/         # Rollback netcode
├── Debug/               # Overlays, sim inspectors, desync tooling
├── Assets/               # Art, audio, fonts (original assets only)
├── Docs/                  # DETERMINISM.md, INPUT.md, COLLISION.md, ROADMAP.md
└── Main.tscn / Main.cs     # Bootstrap scene
```
